  overview: |
    KeyinDot was a custom menu recommendation and community service project built by DND 3rd Team 5.
    It was planned as an Android app where users could share and revisit custom menu combinations that were otherwise scattered across social media.
  background: |
    While discussing topics that the team was commonly interested in, we started talking about custom menu combinations across different brands.
    Custom menus reflect users' preferences and individuality, but choosing many ingredients manually can be difficult, and saving combinations from social media makes them hard to find again.
    KeyinDot aimed to help users discover new combinations and share their own recommendations with others.
  scope: |
    The project consisted of an Android app and a Django-based backend API.
    The implemented scope centered on menu lists, menu details, custom menu creation, image upload, likes, comments, and user authentication.
  contribution: |
    I participated as a backend developer.
    Through one or two regular meetings each week, I joined planning and design discussions, exchanged feedback with the team, and aligned the backend implementation direction with the Android app flow.
  roleDetails:
    - Participated as a Backend Developer
    - Joined weekly or twice-weekly planning and design discussions
    - Strengthened documentation and communication materials for online collaboration
    - Aligned backend implementation direction with the Android app
  implementation: |
    The backend was built with Python 3.7, Django 3.0.2, and Django REST Framework 3.11.0.
    Authentication used JWT, and the menu domain covered custom menus, brands, base menus, ingredients, prices, tips, ratings, likes, comments, images, and tags.

    The Android app was built with Kotlin and communicated with the backend API through Retrofit.
  challenges: |
    The hardest part was that planning, design, and development often interpreted the same feature through different technical assumptions.
    Since the project was mostly online, organizing communication clearly was also difficult.

    To address this, I prepared documentation more carefully and created more supporting materials for explanation.
    At the same time, because this was my first project that spent this much time on in-depth planning and design, I still felt that the code implementation quality could have been stronger.
  outcome: |
    The project was completed as a DND 3rd Team 5 project and was released on Google Play.
    The service is no longer operated and is now closed.
    This experience helped me understand the importance of early planning and collaboration communication more clearly.
