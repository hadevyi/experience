  overview: |
    DB Leap was a junior-year pre-capstone design project focused on controlling a mouse through hand gestures recognized by a Leap Motion sensor.
    At my university, pre-capstone design was treated as a stage that could lead into a graduation project, so we were careful about forming the team.
  background: |
    The project explored how sensor input could be connected to real control actions.
    More than simply building screens, the important part was making the sensor values and user gestures feel connected.
  scope: |
    We built mouse movement, left click, right click, drag, scroll, and detail control flows using hand-gesture input.
    The project focused on receiving sensor input in a desktop application and connecting it to actual mouse control actions.
  contribution: |
    I participated as a team member and worked on the WinForms UI, sensor value tuning, and live demo operation.
    During the presentation, I operated the demo next to the presenter so the features could be shown smoothly.
  roleDetails:
    - Built and organized the WinForms UI according to the demo flow.
    - Adjusted correction criteria for unstable vector values coming from the Leap Motion sensor.
    - Checked that Move, Left Click, Right Click, Drag, Scroll, and Detail Control actions could be demonstrated naturally.
    - Operated the live demo during the presentation.
  implementation: |
    We connected Leap Motion sensor input to mouse control actions.
    Because the incoming sensor values could fluctuate sharply, I adjusted how vector values were normalized before they triggered control actions.
    The UI was organized so each feature could be understood and demonstrated step by step.
  challenges: |
    This was my first project using a sensor, so tracing errors was difficult in itself.
    I had to distinguish whether a problem came from code, sensor input, or gesture thresholds.
    At the time, I did not yet know how to use Git properly, so my UI work and sensor value adjustments were not fully reflected in the Git history.
    Not being able to leave my actual contributions as code history was the biggest regret, and it later made me value collaboration records and documentation more.
  outcome: |
    The project received a strong course evaluation, then led to a department-representative campus competition and later international competitions as a university representative.
    It gave me experience connecting UI work with real sensor input and considering demo stability in a hardware-linked application.
